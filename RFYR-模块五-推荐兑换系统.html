关键设计

邀请奖励分两类：非会员奖励计入 bonus_read_count（终身有效，不消耗则不消失）；会员奖励计入 bonus_daily_count（每日北京时间重置）。这种设计兼顾了激励效果与平台成本控制。

02 奖励规则
激励机制 可配置

邀请奖励每次成功邀请获得阅读配额

用户通过分享带 ?ref=xxx 参数的链接邀请新用户注册。被邀请人成功注册后，邀请人获得奖励阅读次数。

邀请人身份奖励字段有效期重置方式
非会员bonus_read_count终身不重置，持续累加
月卡会员bonus_daily_count当日（CST）北京时间每日凌晨自动归零
年卡/永久会员bonus_daily_count当日（CST）每日归零（年卡本身无阅读限制，奖励暂不消费）
默认配置

平台级配置项 referral_bonus_count 决定每次邀请奖励多少篇阅读，默认值为 2 篇/每人。管理员可通过 reading_settings 表动态调整，无需发版。

兑换码规则月卡/年卡兑换时限与次数限制

⏱️ 兑换码有效期：生成后 3 天内必须使用，过期后状态自动变为 expired
📅 月卡时长：30 天，到期后状态变为 expired
🗓️ 年卡时长：365 天（精确一年，使用 setFullYear 计算，避免闰年误差）
🚫 自兑限制：用户不能使用自己生成的兑换码，防止刷号套利
💳 免费月卡：每人限 1 次（通过 monthly_free_used 标记）
🛒 付费月卡：每人最多 4 次（通过 monthly_purchase_count 计数）

续期逻辑已有会员顺延，非覆盖

当用户再次使用兑换码时，系统采用「顺延」策略：新时长从当前会员到期日的次日开始计算，而非从当天覆盖。这一逻辑在 addMembershipPeriod 函数中实现：

typescript lib/redeem.ts
// lib/redeem.ts — 续期日期计算 export function addMembershipPeriod( baseDate: Date, days: number, membershipType: string ): Date { const result = new Date(baseDate); if (membershipType === 'yearly') { result.setFullYear(result.getFullYear() + 1); } else { result.setDate(result.getDate() + days); } return result; }
边界处理

JavaScript 的 setDate 会自动处理跨月（如 1月31日 + 30天 → 3月2日），无需手动判断月份天数。

buildReferralChain推荐链构建

向上追溯推荐链，返回指定用户的所有上游推荐人 ID 列表（最深 3 层），用于检测循环引用和多账号套利攻击。

typescript lib/referral.ts — buildReferralChain
// lib/referral.ts export async function buildReferralChain( supabase: SupabaseClient, startUserId: string, maxDepth: number = 3 ): Promise<string[]> { const chain: string[] = []; let currentId: string | null = startUserId; for (let i = 0; i < maxDepth; i++) { const { data } = await supabase .from('referrals') .select('referrer_id') .eq('referee_id', currentId!) .single(); if (!data) break; chain.push(data.referrer_id); currentId = data.referrer_id; } return chain; }

generateRedeemCode密码学安全码生成

兑换码使用 Node.js 内置 crypto.randomBytes 生成，而非 Math.random()，确保不可预测性，防止被枚举攻击。

typescript lib/redeem.ts — generateRedeemCode
// lib/redeem.ts const CHARS = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'; export function generateRedeemCode(type: string): string { const prefix = type === 'monthly' ? 'RFYR-MONTH-' : 'RFYR-YEAR-'; const bytes = crypto.randomBytes(6); let code = ''; for (let i = 0; i < 6; i++) { code += CHARS[bytes[i] % CHARS.length]; } return prefix + code; }
安全注意

去掉了 I、O、0、1 等易混淆字符，最终字符集大小为 30，每 6 位组合数为 30⁶ ≈ 7.29 亿，暴力枚举在计算上不可行。

速率限制/api/referral/validate 的双重防线

🌐 IP 级别：同一 IP 在 60 秒内最多 20 次验证请求，超限返回 429 Too Many Requests
👤 用户级别：同一用户 ID 在 60 秒内最多 50 次验证请求
🔄 降级策略：优先使用 rate_limits 表，若数据库写入失败则降级使用内存 Map 记录，防止拒绝服务
🧹 异步清理：V-L-06 修复：限流记录清理改为异步，不阻塞主流程

安全原则

推荐和兑换系统涉及真实货币（会员订阅）和内容权益（阅读配额），安全性至关重要。核心原则：随机数必须密码学安全、所有外部输入必须白名单验证、关键操作必须原子化。
