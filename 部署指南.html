第一阶段 · 连接服务器
1
用 SSH 连接到服务器
SSH 就是远程控制服务器的工具，像是用键盘操作服务器的终端
约 2 分钟
服务器 IP43.155.237.163
用户名root
端口22（默认）
如果你用的是 Mac

打开「终端」（按 Command+空格 搜索"终端"），输入以下命令：

ssh root@43.155.237.163
# 第一次连接会问你是否信任，输入 yes 回车
# 然后输入密码（输入时屏幕不显示字符，属于正常现象，输完回车）
如果你用的是 Windows

方法一（推荐）：打开「Windows 终端」或「PowerShell」，和 Mac 一样输入：

ssh root@43.155.237.163

方法二：下载安装 PuTTY（免费软件），Host Name 填 43.155.237.163，点 Open，输入用户名 root 和密码。

✅
连接成功后，终端会显示类似 root@VM-xxx-xxx:~# 的提示符，说明你已经进入服务器了。
2
找到并登录宝塔面板
宝塔是服务器的图形化管理界面，用浏览器打开
约 3 分钟
① 在服务器终端里运行这条命令

SSH 连进服务器后，输入：

bt default
# 这会输出宝塔面板的地址和初始账号密码

输出大概长这样：

======================== 面板账号登录信息 ========================
外网面板地址: http://43.155.237.163:8888/xxxxxxxx
内网面板地址: http://10.0.x.x:8888/xxxxxxxx
username: xxxxxxxx
password: xxxxxxxx
==================================================================
② 用浏览器打开面板

复制上面输出的「外网面板地址」，粘贴到浏览器打开，用显示的账号密码登录。

⚠️
如果浏览器打不开，需要去腾讯云控制台 → 安全组 → 放行 8888 端口（TCP，来源填你自己的 IP）。
③ 登录后修改密码（重要）

宝塔面板右上角 → 头像 → 修改密码，设置一个安全的新密码并记下来。

第二阶段 · 安装必要软件
3
在宝塔面板安装 Nginx、Node.js、PM2
点几下鼠标即可，宝塔会自动下载安装
约 10 分钟
宝塔面板 → 软件商店，依次安装：
  • ☐
    Nginx — 选稳定版（如 1.24），点安装，等待完成
  • ☐
    Node.js 版本管理器 — 搜索"Node"找到它，安装后，点进去选择 20.x LTS 版本安装
  • ☐
    PM2 管理器 — 搜索"PM2"，点安装（PM2 是让程序一直在后台运行的工具）
安装 Git（在终端里执行）

宝塔面板没有 Git 的图形化安装入口，需要在终端里用一条命令安装：

yum install -y git
# 出现 Complete! 说明安装成功

# 验证安装
git --version
# 应显示 git version 2.x.x
验证 Node.js 和 PM2
node -v    # 应显示 v20.x.x
npm -v     # 应显示 10.x.x
pm2 -v     # 应显示版本号
⚠️
如果 node -v 提示找不到命令，运行 source /etc/profile 再试一次。
第三阶段 · 配置代码仓库
4
本地代码推送到 GitHub
GitHub 是代码的中转站，本地 → GitHub → 服务器
约 10 分钟
① 确保 .env 文件不被上传（在你的电脑上做）
🚫
必须检查：.env.production 里有数据库密钥，绝对不能传到 GitHub（即使是私有仓库也建议不传）。

在你的电脑上，打开项目里的 .gitignore 文件，确认有这几行（没有就加上）：

.env.production
.env.local
.env*.local
② 在 GitHub 新建私有仓库，推送代码

去 github.com，点右上角「+」→「New repository」，名字填 rfyr，选 Private（私有），创建。

然后在你的电脑终端里，进入项目目录执行：

# 添加 GitHub 远程仓库（把"你的用户名"改成你的 GitHub 用户名）
git remote add origin git@github.com:你的用户名/rfyr.git

# 推送代码
git add -A
git commit -m "init: 初始化项目"
git push -u origin main
③ 服务器生成 SSH Key，授权访问 GitHub

服务器需要"通行证"才能从 GitHub 拉取代码。在服务器终端里执行：

# 生成 SSH 密钥（一路回车，不需要设密码）
ssh-keygen -t ed25519 -C "rfyr-server" -f ~/.ssh/id_ed25519

# 查看公钥，复制全部输出的内容
cat ~/.ssh/id_ed25519.pub

复制输出的内容（以 ssh-ed25519 开头的那一行），然后：

打开 GitHub → 右上角头像 → Settings → 左侧 SSH and GPG keys → New SSH key → 粘贴进去，标题随便填，保存。

# 测试是否成功，看到 "Hi xxx! You've successfully authenticated" 就对了
ssh -T git@github.com
④ 服务器拉取代码
# 进入网站目录
cd /www/wwwroot/

# 克隆代码（把"你的用户名"改成你的 GitHub 用户名）
git clone git@github.com:你的用户名/rfyr.git rfyr

# 进入项目
cd rfyr
ls   # 能看到 app/ backend/ package.json 等文件就说明成功
第四阶段 · 配置环境变量
5
在服务器上创建环境变量文件
把 Supabase、七牛云等密钥告诉程序，这些文件不在 Git 里，需要手动创建
约 5 分钟
💡
这一步只需要做一次。以后更新代码，这个文件不会被覆盖。nano 是服务器上的文本编辑器，用法：编辑完按 Ctrl+O 保存，按 Ctrl+X 退出。
① 创建前端环境变量（.env.production）
nano /www/wwwroot/rfyr/.env.production

把下面内容粘贴进去，把黄色部分替换成你的实际值：

# ── Supabase（在 Supabase 控制台 → Settings → API 里找）──────────
NEXT_PUBLIC_SUPABASE_URL=https://你的项目ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的 anon public key
SUPABASE_SERVICE_ROLE_KEY=你的 service_role key（不能泄露）

# ── 管理员邮箱 ──────────────────────────────────────────────────
ADMIN_EMAILS=zhoujia0718@163.com

# ── Resend 邮件（在 resend.com 控制台找）────────────────────────
RESEND_API_KEY=re_xxxxxxxxxxxxxxxx
RESEND_SENDER_EMAIL=noreply@rfyr.club

# ── Token 签名密钥（必须和本地 .env.local 里的值一致）───────────
HMAC_SECRET=rfyr_你的密钥字符串

# ── 七牛云（在七牛云控制台 → 密钥管理 里找）────────────────────
QINIU_ACCESS_KEY=你的 AccessKey
QINIU_SECRET_KEY=你的 SecretKey
QINIU_BUCKET=你的 bucket 名称
QINIU_DOMAIN=https://你的七牛加速域名

编辑完后：按 Ctrl+O → 回车 → 按 Ctrl+X

② 创建后端环境变量（backend/.env）
nano /www/wwwroot/rfyr/backend/.env
PORT=3001
SUPABASE_URL=https://你的项目ID.supabase.co
SUPABASE_SERVICE_ROLE_KEY=你的 service_role key

同样 Ctrl+O 保存，Ctrl+X 退出。

环境变量在哪里找
变量在哪里找必填
NEXT_PUBLIC_SUPABASE_URLSupabase 控制台 → Settings → API → Project URL必填
NEXT_PUBLIC_SUPABASE_ANON_KEY同上页面 → anon public必填
SUPABASE_SERVICE_ROLE_KEY同上页面 → service_role（点眼睛图标显示）必填
HMAC_SECRET从你本地的 .env.local 文件里复制必填
QINIU_ACCESS_KEY / SECRET_KEY七牛云控制台 → 右上角头像 → 密钥管理必填
QINIU_BUCKET七牛云控制台 → 对象存储 → 你的 bucket 名字必填
QINIU_DOMAIN七牛云 → bucket → 域名管理 → 加速域名（带 https://）必填
RESEND_API_KEYresend.com → API Keys选填
第五阶段 · 构建并启动
6
安装依赖并构建项目
npm install 下载所有库，npm run build 把代码编译成可运行的版本
约 5~8 分钟
# 进入项目目录
cd /www/wwwroot/rfyr

# 安装前端的所有依赖库（会下载很多东西，耐心等待）
npm install

# 构建 Next.js（把代码编译成生产版本，需要 3-5 分钟）
npm run build

# 安装后端的依赖库
cd backend && npm install && cd ..
✅
npm run build 完成后，如果看到 Route (app) 表格和 ✓ Compiled 之类的输出，就说明构建成功了。
⚠️
如果 build 报错，大概率是环境变量没填对。检查第5步的 .env.production 文件，确认没有多余空格或漏填的值。
7
用 PM2 启动前端和后端
PM2 让程序一直在后台运行，服务器重启后自动恢复
约 3 分钟
① 更新 PM2 配置文件，加入后端进程
nano /www/wwwroot/rfyr/ecosystem.config.cjs

把文件内容全部替换为：

module.exports = {
  apps: [
    {
      name: 'rfyr-web',                        // 前端 Next.js，端口 3000
      cwd: '/www/wwwroot/rfyr',
      script: 'node_modules/next/dist/bin/next',
      args: 'start',
      instances: 1,
      autorestart: true,
      max_restarts: 10,
      env: { NODE_ENV: 'production', PORT: '3000' },
    },
    {
      name: 'rfyr-api',                        // 后端 Express，端口 3001
      cwd: '/www/wwwroot/rfyr/backend',
      script: 'server.js',
      instances: 1,
      autorestart: true,
      max_restarts: 10,
      env: { NODE_ENV: 'production', PORT: '3001' },
    },
  ],
}
② 启动所有进程
cd /www/wwwroot/rfyr

# 清除旧进程（如果有的话）
pm2 delete all 2>/dev/null; true

# 启动
pm2 start ecosystem.config.cjs

# 查看状态 → 两个应用都应该显示绿色的 online
pm2 status

正常输出长这样：

┌─────┬───────────┬─────────┬──────┬───────┬────────┐
│ id  │ name      │ mode    │ ↺    │ status│ cpu    │
├─────┼───────────┼─────────┼──────┼───────┼────────┤
│ 0   │ rfyr-web  │ fork    │ 0    │ online │ 0%     │
│ 1   │ rfyr-api  │ fork    │ 0    │ online │ 0%     │
└─────┴───────────┴─────────┴──────┴───────┴────────┘
③ 设置开机自启（非常重要）
# 保存当前进程列表
pm2 save

# 生成开机启动命令（会输出一条 sudo 开头的命令）
pm2 startup
# ↑ 复制它输出的那条命令，粘贴执行（如果已经是 root 用户直接执行即可）
第六阶段 · 配置域名和 HTTPS
8
宝塔建站 + 配置 Nginx 反向代理
让域名指向你的网站，Nginx 负责把请求转发给 Next.js
约 5 分钟
① 先在域名服务商添加 DNS 解析

去你购买域名的地方（如阿里云、腾讯云），添加以下 DNS 记录：

记录类型主机名记录值
A@（或留空）43.155.237.163
Awww43.155.237.163
⏳
DNS 解析生效通常需要 5~30 分钟，先继续做下面的步骤，不用等。
② 宝塔面板 → 网站 → 添加站点
  • ☐域名填你的域名，如 rfyr.club 和 www.rfyr.club（一行一个）
  • ☐根目录填 /www/wwwroot/rfyr
  • ☐PHP 版本选 纯静态
  • ☐点击确认创建
③ 修改 Nginx 配置文件

网站列表 → 点击你的域名 → 设置 → 配置文件，把里面的内容全部清空，替换为以下内容（把域名改成你自己的）：

server {
    listen 80;
    server_name rfyr.club www.rfyr.club;

    # 支持最大 200MB 的文件上传（PDF 书籍）
    client_max_body_size 200M;

    # 所有请求转发给 Next.js（端口 3000）
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}

点 保存，Nginx 会自动重载。

✅
此时访问 http://rfyr.club（不带 s）应该能看到网站了。
9
申请 SSL 证书，开启 HTTPS
免费证书，宝塔一键申请，浏览器地址栏显示绿色锁
约 2 分钟
  • ☐宝塔面板 → 网站 → 点击域名 → SSL
  • ☐选择 Let's Encrypt，勾选所有域名
  • ☐点 申请，等待 30 秒出现「证书申请成功」
  • ☐开启右上角的 强制 HTTPS 开关
🎉
完成！现在访问 https://rfyr.club 就能看到你的网站，地址栏有绿色锁。
🔄
Let's Encrypt 证书 90 天到期，宝塔会自动续签，无需手动操作。
10
配置防火墙（收尾工作）
只开放必要端口，保护服务器安全
约 3 分钟
腾讯云控制台 → 安全组 → 入站规则

确保以下端口的状态：

端口协议来源 IP操作
22TCP你的 IP（或 0.0.0.0/0）保留（SSH 用）
80TCP0.0.0.0/0保留（HTTP）
443TCP0.0.0.0/0保留（HTTPS）
8888TCP只填你自己的 IP宝塔面板入口
3000、3001TCP—不要开放
🔐
端口 3000 和 3001 是程序内部端口，不要对外开放，只通过 Nginx 代理访问。宝塔面板端口 8888 也建议只开放给你自己的 IP。
日常维护 · 更新代码
↺
以后如何更新代码
本地改好 → push 到 GitHub → 服务器一键部署
每次约 5 分钟
第一步：在你的电脑上提交并推送
git add -A
git commit -m "描述你改了什么"
git push origin main
第二步：创建部署脚本（只需做一次）
# 在服务器上执行，创建一个快捷部署脚本
cat > /www/wwwroot/rfyr/deploy.sh << 'EOF'
#!/bin/bash
set -e
cd /www/wwwroot/rfyr
echo "⬇️  拉取最新代码..."
git pull origin main
echo "📦 安装依赖（如有新增）..."
npm install
echo "🏗️  重新构建..."
npm run build
echo "🔄 重启服务..."
pm2 restart rfyr-web
pm2 restart rfyr-api
echo "✅ 部署完成！"
EOF
chmod +x /www/wwwroot/rfyr/deploy.sh
第三步：以后每次更新只需执行
cd /www/wwwroot/rfyr && ./deploy.sh
💡
如果只改了 backend/ 里的代码，不需要重新 build，只需 git pull && pm2 restart rfyr-api，快很多。
