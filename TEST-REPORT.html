4. 修复的问题

TypeScript 编译错误修复

✅ FIX-001: app/actions/auth.ts - 缺少 hashVerificationCodeSync 导入

问题：函数使用了 hashVerificationCodeSync 但未导入

修复：添加 `import { hashVerificationCodeSync } from '@/lib/security'`

✅ FIX-002: components/payment-dialog.tsx - 无效的会员类型 'weekly'

问题：`activateMembership('weekly', 7)` 使用了无效的类型

修复：改为 `activateMembership('monthly', 30)` (月卡30天)

✅ FIX-003: lib/reading-limit.ts - 重复导出 resolveUserIdFromRequest

问题：函数定义后又被 export，导致 TS2484 错误

修复：移除重复的 export 语句

✅ FIX-004: lib/referral.ts - 未定义的 profile 变量

问题：使用未定义的 `profile` 变量获取邀请人奖励

修复：添加查询获取 referrerProfile 后再使用

运行时错误修复

✅ FIX-005: app/notes/[slug]/page.tsx - guest_read_limit 未定义

问题：buildArticlePage 函数无法访问 React hooks 中的变量

修复：将 guest_read_limit 和 monthly_daily_limit 作为参数传入函数

✅ FIX-006: app/notes/page.tsx - 使用不存在的 isLoggedIn 属性

问题：useMembership() 返回类型中没有 isLoggedIn 属性

修复：使用 `membershipType !== "none"` 判断登录状态

✅ FIX-007: .next 缓存目录损坏

问题：构建清单文件缺失导致 Internal Server Error

修复：删除 .next 目录，重新构建
